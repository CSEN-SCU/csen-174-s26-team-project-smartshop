## Sprint 1 Retro Write-up
### What went well in Sprint 1?

What went well for our team was that we had a pretty clear division of work. Each person had a specific area, like frontend, backend, AI integration, or database, so we were not all trying to work on the exact same thing. This helped us move faster because everyone could focus on their own part.

Another thing that went well was that we were able to get real tests into the repo instead of only talking about them in the write-up. Some tests turned green, while the unfinished feature tests stayed red on purpose, which matched the TDD idea. We also helped each other check for overlap and made sure the tests connected back to our SmartShop features.

We should celebrate that we made progress as a team even though the merge process and test organization were a little confusing. We still got the main pieces together: tests, skill setup, write-up, and Jolli connection. Overall, the team handled the assignment well by dividing the work and communicating when something needed to be checked.

### What could be improved?

I think one thing we could improve is the merge process. Everyone was working on tests in different areas, but we did not fully decide where the final test files should go or who should merge them. Because of that, we had some confusion with branches, PR conflicts, and whether tests were overlapping.

Another issue was that our roles were clear in general, but not clear enough for testing. For example, backend, frontend, AI, and database tests can connect to each other, so we had to check after the fact if someone’s tests overlapped.

### Which improvements will the team commit to in Sprint 2?

Next time, we should first agree on the test folder structure, then list who owns which tests before generating or committing them. We should also have one person coordinate PRs and merge conflicts, and one person handle setup tasks like Jolli or CI/CD.

We will dedicate the first 30 minutes of lab to make sure we are all on the same page. We will discuss who has done what and assign tasks. This will give everyone a clear idea of what their tasks are and how everything will come together.
